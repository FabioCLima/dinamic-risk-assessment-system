This is where the project needs the clearest decision rule. The rubric language points toward comparing model quality using F1, with “better” meaning a higher score than what is currently deployed. In practice, the easiest fix is to make fullprocess.py follow this exact sequence: check for new files and exit immediately if none are found, ingest and train a candidate model only when new files exist, score that candidate model, read the deployed baseline score from production_deployment/latestscore.txt, and then deploy only when candidate_score > deployed_score. If no deployed score exists yet, treat that as an initial deployment case and allow deployment once.

# 1) Stop early if no new data
if len(new_files) == 0:
    print("No new data found. Process ends here.")
    return

# 2) Build candidate model from new data
ingestion.merge_multiple_dataframe()
training.train_model()
candidate_score = scoring.score_model()

# 3) Compare with deployed baseline
deployed_score = read_deployed_score()  # from production_deployment/latestscore.txt
if deployed_score is None:
    should_deploy = True   # first deployment case
else:
    should_deploy = candidate_score > deployed_score

# 4) Deploy only when improvement is confirmed
if should_deploy:
    deployment.store_model_into_pickle()

This makes the pass/fail condition observable and avoids accidental redeployments when the candidate model is equal to or worse than the deployed one.
