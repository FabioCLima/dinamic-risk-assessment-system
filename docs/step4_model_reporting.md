# Step 4 — Model Reporting (Reporting + API)

Model reporting is important because it helps data scientists stay aware of all aspects of the data, the model, the training processes, and performance. Automated reporting also enables you to keep stakeholders quickly and reliably informed.

In this step, you’ll write scripts that create reports related to your ML model, its performance, and related diagnostics.

## Starter files

You will work with:
- `reporting.py` (generate plots)
- `app.py` (API setup)

## 1) Generating plots (`reporting.py`)

Update `reporting.py` to generate plots related to the ML model’s performance.

### Inputs

- Use the **model prediction function** you created in `diagnostics.py` (Step 3).
- Use the **test data** in `test_data_path` (from `config.json`) as the input dataset.

### Output

Generate a **confusion matrix** plot and save it as:

- `confusionmatrix.png`

Save the image to the directory specified in `output_model_path` (from `config.json`).

## 2) API setup (`app.py`)

Set up an API so that you and your colleagues can access ML diagnostics and results.

### Requirements

- Provide **four** endpoints:
  - one for model predictions
  - one for model scoring
  - one for summary statistics
  - one for other diagnostics
- Each endpoint must return an **HTTP 200** status code.

### Endpoints

#### `/prediction`

- Input: a dataset file location
- Output: predictions from the prediction function created in Step 3

#### `/scoring`

- Behavior: run the `scoring.py` logic from Step 2
- Output: the model score (F1)

#### `/summarystats`

- Behavior: run the summary statistics function created in Step 3
- Output: summary statistics values

#### `/diagnostics`

- Behavior: run the timing, missing-data, and dependency-check functions created in Step 3
- Output: diagnostic values

## 3) Calling your API endpoints (`apicalls.py`)

Use `apicalls.py` to call each endpoint, combine the outputs, and write them to:

- `apireturns.txt`

### Notes

- For the `/prediction` endpoint, you can use `/testdata/testdata.csv` as input (predictions on the test set).
- For the other endpoints, you don’t need to specify any inputs.
- Save `apireturns.txt` to the directory specified in `output_model_path` (from `config.json`).
